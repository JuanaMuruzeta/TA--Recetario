# Buscador de Recetas## Integrantes- Juana Muruzeta- Delfina Fernandez Covaro- Regina Francucci- Giulia Maniotti- Conrado Soldo---## Objetivo del proyectoEl objetivo del proyecto es desarrollar un sistema en Python que permita buscar recetas de cocina a partir de ingredientes ingresados por el usuario. El programa utiliza información obtenida desde una API externa, procesa los datos recibidos y muestra distintas opciones de recetas disponibles.El sistema busca facilitar la elección de comidas permitiendo consultar recetas, conocer sus ingredientes e instrucciones de preparación, además de almacenar las búsquedas realizadas y generar gráficos con la información obtenida.---## Descripción general del funcionamientoAl iniciar el programa, el usuario visualiza un menú principal con diferentes opciones:1. Buscar recetas por ingrediente.2. Ver detalle de una receta.3. Consultar historial de búsquedas.4. Generar gráficos/resumen.5. Salir del programa.El usuario ingresa uno o más ingredientes y el sistema realiza una búsqueda mediante la API TheMealDB. Luego procesa la información recibida y muestra las recetas encontradas.Cada búsqueda realizada se almacena dentro de un historial para poder consultarla posteriormente.---## Principales funcionalidades- Buscar recetas según ingredientes ingresados por el usuario.- Conectarse con una API externa para obtener información.- Mostrar información detallada de cada receta.- Guardar un historial de búsquedas realizadas.- Leer información previamente almacenada.- Generar gráficos a partir de los datos obtenidos.- Utilizar un menú interactivo mediante consola.---## Distribución del trabajo- Juana Muruzeta: - Delfina Fernandez Covaro: - Regina Francucci: - Giulia Maniotti: - Conrado Soldo: ---# Fuente de datos utilizadaEl proyecto utiliza la API pública TheMealDB:https://www.themealdb.com/api.phpTheMealDB es una base de datos online que contiene información sobre recetas de distintos países.Información utilizada:- Nombre de la receta.- ID de la receta.- Ingredientes.- Instrucciones de preparación.- Categoría.- Tipo de cocina / Pais de porvenencia.- Imagen de la receta.Consultas realizadas:- Búsqueda de recetas por ingrediente.- Obtención del detalle completo de una receta mediante su ID.---# Instrucciones para ejecutar el programa1. Ingresar al menu2. Seleccionar la opcion que desees (1. Buscar recetas por ingredientes, 2. Ver historial de búsquedas, 3.Ver gráficos del historial, Salir)3. Buscar recetas por ingredientes: Si el usuario elige la opción 1, el programa le pide que ingrese ingredientes separados por coma.4. Después de ingresar ingredientes, el programa pregunta si quiere aplicar un filtro por país.4. Ver historial: Si el usuario elige la opción 2 del menú, el programa lee el CSV y muestra el historial.5. Ver gráficos del historial: Si el usuario elige la opción 3, el programa genera gráficos usando la información guardada en el historial o del API.6. Salir: si elige opcion 4 el programa termina---# Librerías utilizadas## requestsPermite conectarse con la API TheMealDB y obtener la información de las recetas en formato JSON.## matplotlibSe utiliza para crear gráficos con la información obtenida.## jsonPermite guardar y recuperar datos del historial de búsquedas.## osSe utiliza para trabajar con archivos dentro del proyecto.---# Estructura del repositorio```TA--Recetario/│├── src/││   ├── main.py│   ├── api_recetas.py│   ├── historial.py│   └── graficos.py│├── README.md└── requirements.txt```Descripción:**main.py**Archivo principal del proyecto. Chicken Curry2. Chicken Soup3. Chicken Pasta```Ejemplo detalle:```Nombre: Chicken CurryCategoría: ChickenIngredientes:- Chicken- Curry- OnionInstrucciones:Preparar los ingredientes siguiendo los pasos indicados.```---# Diagramas de diseñoEl proyecto incluye diagramas de flujo que representan el funcionamiento del sistema.Diagramas realizados:- Diagrama general del programa.- Diagrama del main.py.- Diagramas individuales de las funciones principales.Los diagramas muestran:- Inicio y fin del programa.- Decisiones tomadas por el usuario.- Llamado entre funciones.- Flujo de información.---# Declaración de uso de Inteligencia ArtificialDurante el desarrollo del proyecto se utilizó Inteligencia Artificial (ChatGPT) como herramienta de apoyo.La IA fue utilizada para:- Organizar la estructura del programa.- Revisar código.- Mejorar documentación.- Ayudar en la creación de diagramas de flujo.- Corregir errores y optimizar funciones.El desarrollo, pruebas y decisiones finales fueron realizadas por los integrantes del grupo.---# Notas adicionales- Es necesario contar con conexión a internet para obtener las recetas desde la API.- Si ocurre un problema de conexión, el programa informa el error y continúa funcionando.- Se recomienda ejecutar el programa desde el archivo main.py.
