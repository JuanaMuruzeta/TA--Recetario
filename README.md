# Buscador de Recetas## Integrantes* Juana Muruzeta* Delfina Fernandez Covaro* Regina Francucci* Giulia Maniotti* Conrado Soldo---# Objetivo del proyectoEl objetivo del proyecto es desarrollar un sistema interactivo en Python que permita recomendar recetas de cocina a partir de ingredientes ingresados por el usuario.El programa utiliza información obtenida desde la API pública TheMealDB y aplica procesamiento propio para calcular coincidencias entre los ingredientes ingresados y los ingredientes de cada receta encontrada.Además, el sistema permite filtrar recetas por país, visualizar detalles completos de cada receta, almacenar un historial de búsquedas y generar gráficos estadísticos a partir de la información obtenida.---# Descripción general del funcionamientoAl iniciar el programa, el usuario visualiza un menú principal con las siguientes opciones:1. Buscar recetas por ingredientes.2. Ver historial de búsquedas.3. Ver gráficos del historial.4. Salir.Cuando el usuario realiza una búsqueda:* Ingresa uno o más ingredientes separados por coma.* Puede aplicar un filtro opcional por país.* El programa consulta la API TheMealDB.* Obtiene las recetas encontradas.* Calcula la cantidad de ingredientes coincidentes.* Calcula el porcentaje de coincidencia para cada receta.* Ordena las recetas según el porcentaje de coincidencia.* Muestra un ranking de resultados.* Permite consultar el detalle completo de cada receta.* Guarda la búsqueda en el historial.---# Principales funcionalidades* Búsqueda de recetas por ingrediente.* Filtro opcional por país o región.* Conexión con la API TheMealDB.* Obtención de información detallada de cada receta.* Cálculo de ingredientes coincidentes.* Cálculo del porcentaje de coincidencia.* Ranking de recetas ordenadas según coincidencia.* Almacenamiento de historial de búsquedas.* Consulta del historial.* Generación de gráficos estadísticos.* Menú interactivo mediante consola.* Manejo de errores y validación de entradas.---# Distribución del trabajo* Juana Muruzeta: procesamiento de datos, cálculo de coincidencias y porcentajes.* Delfina Fernandez base de datos online que contiene recetas de distintos países y categorías gastronómicas.Información utilizada:* Nombre de la receta.* ID de la receta.* Ingredientes.* Instrucciones de en distintos módulos para mejorar la estructura, reutilización y mantenimiento del código.---# Funciones principales## buscar_recetas_por_ingrediente()Busca recetas utilizando un ingrediente ingresado por el usuario.Retorna una lista de recetas encontradas.---## obtener_detalle_receta()Obtiene la información completa de una receta utilizando su identificador.Permite acceder a:* Ingredientes.* Instrucciones.* Categoría.* País de origen.---## procesar_json_recetas_api()Procesa los datos obtenidos desde la API y genera una estructura más adecuada para el programa.Calcula:* Coincidencias.* Porcentaje de coincidencia.* Categoría.* País.* Ingredientes.* Instrucciones.---## ordenar_por_coincidencia()Ordena las recetas desde la de mayor coincidencia hasta la de menor coincidencia.---## filtrar_recetas_por_pais()Permite mostrar únicamente las recetas pertenecientes al país seleccionado por el usuario.---## guardar_busqueda()Guarda la búsqueda realizada por el usuario dentro del historial.---## mostrar_historial()Muestra el historial completo de búsquedas realizadas.---## generar_grafico_historial()Genera los gráficos del proyecto utilizando la información almacenada en el historial.Incluye:* Ingredientes más buscados agrupados por categoría.* Países más solicitados.* Mejor porcentaje de coincidencia por búsqueda.---## main()Función principal encargada de controlar el flujo general del programa.Gestiona:* Menú principal.* Interacción con el usuario.* Búsquedas.* Historial.* Gráficos.* Finalización del sistema.---# Resultados y funcionalidades generadasEl programa genera distintas salidas para el usuario:* Ranking de recetas encontradas.* Cantidad de resultados obtenidos.* Porcentaje de coincidencia por receta.* Detalle completo de recetas.* Historial de búsquedas.* Gráficos estadísticos.Ejemplo:```textIngrese ingredientes: pollo, arrozRECETAS ENCONTRADAS1. Chicken Rice Bowl   País: Canadian   Coincidencias: 2   Porcentaje: 100%```Ejemplo de detalle:```textNombre: Chicken Rice BowlPaís: CanadianIngredientes:- Chicken- Rice- OnionInstrucciones:Preparar los ingredientes siguiendo los pasos indicados.```---# Diagramas de diseñoEl proyecto incluye diagramas de flujo que representan el funcionamiento general del sistema.Diagramas incluidos:* Diagrama general del programa.* Diagrama del flujo principal.* Diagramas de las funciones principales.Los diagramas muestran:* Flujo del menú.* Interacción del usuario.* Comunicación entre módulos.* Procesamiento de datos.* Generación de resultados.---# Declaración de uso de Inteligencia ArtificialDurante el desarrollo del proyecto se utilizó ChatGPT como herramienta de apoyo.La IA fue utilizada para:* Diseñar la estructura general del sistema.* Organizar módulos y funciones.* Revisar código.* Detectar errores.* Mejorar la documentación.* Asistir en la elaboración de diagramas de flujo.Todas las decisiones finales de implementación, pruebas y validación fueron realizadas por los integrantes del grupo.---# Notas adicionales* Es necesario contar con conexión a internet para acceder a la API TheMealDB.* Si ocurre un problema de conexión, el programa informa el error mediante mensajes al usuario.* Los gráficos generados se almacenan automáticamente dentro de la carpeta `graficos/`.* El historial de búsquedas se almacena en el archivo `datos/historial_busquedas.csv`.* Se recomienda ejecutar el programa desde el archivo `main.py`.
